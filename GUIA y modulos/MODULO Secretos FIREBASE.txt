MODULO SECRETOS - DISENO FUNCIONAL + TECNICO (MVP -> ESCALA)
===============================================================================

===============================================================================
0) OBJETIVO DEL MODULO
===============================================================================
Crear un espacio:
- anonimo
- viral
- comunitario

Que se sienta:
- mas crudo que Noticias
- mas libre que Comunidad
- mas adictivo que Encuestas

Objetivo de producto:
- "quiero leer uno mas"
- "esto es real"
- "esto no lo veo en otro lado"


===============================================================================
1) EXPERIENCIA BASE (QUE VE EL USUARIO)
===============================================================================
Pantalla "Secretos":
1. Destacados arriba:
   - Top secretos del dia
   - Mas comentados
   - Mas votados

2. Stream principal:
   - texto del secreto
   - votos positivos/negativos (+ / -)
   - cantidad de comentarios
   - tiempo relativo (hace 2h)
   - etiqueta opcional (zona / categoria)
   - tendencia (subiendo / bajando / estable)

3. Filtros simples:
   - Recientes
   - Populares
   - Polemicos (sin resolver)
   - Por zona

Microcopy recomendado:
- "Comparte algo anonimo y real."
- "No publiques nombres ni insultos."
- "Describe un hecho, no acuses directamente."


===============================================================================
2) UX DE PUBLICACION (SIN FRICCION)
===============================================================================
Formulario "Crear secreto":
- Campo obligatorio:
  * texto (max 280 chars recomendado, min 12)

- Campos opcionales:
  * sexo: No responder (default), Hombre, Mujer
  * edad
  * categoria
  * zona

Reglas UX:
- nada de registro obligatorio para publicar
- validacion en vivo de caracteres
- bloquear texto vacio y texto solo emojis
- publicar en 1 paso
- mostrar estado anonimo visible ("Publicaras en modo anonimo")


===============================================================================
3) MODELO DE DATOS (ALINEADO A TU ARQUITECTURA)
===============================================================================
Coleccion principal (recomendada, unificada):
- content/{contentId}

Secreto en content:
- module: "secrets"
- type: "secret"
- source: "anonymous"
- isOficial: false
- titulo: "" (opcional vacio, o no usar)
- descripcion: "<texto secreto>"
- category: "rumores" | "relaciones" | "trabajo" | "denuncia_light" | "random" | null
- zone: "<barrio o zona>" | null
- sex: "no_responder" | "hombre" | "mujer"
- age: number | null
- stats:
  * upVotesCount: number
  * downVotesCount: number
  * commentsCount: number
  * reportsCount: number
  * viewsCount: number
- rank:
  * score: number
  * hotScore: number
  * controversyScore: number
  * trend: "up" | "down" | "stable"
- moderation:
  * status: "active" | "hidden_auto" | "hidden_admin" | "blocked"
  * reason: string | null
  * reviewedBy: string | null
  * reviewedAt: timestamp | null
- createdAt, updatedAt, deletedAt

Subcolecciones:
- content/{contentId}/secret_votes/{fingerprintHash}
  * vote: 1 | -1
  * createdAt, updatedAt

- content/{contentId}/secret_comments/{commentId}
  * text
  * anonAlias
  * createdAt, updatedAt, deletedAt
  * score (para relevancia)
  * reportsCount

- content/{contentId}/secret_reports/{fingerprintHash}
  * reason
  * createdAt

Colecciones de control anti-spam:
- secret_rate_limits/{fingerprintHash}
  * lastSecretAt
  * dailyCount
  * dailyWindowStart
  * expiresAt (TTL)

- secret_fingerprints/{fingerprintHash}
  * firstSeenAt
  * lastSeenAt
  * expiresAt (TTL 72h)

IMPORTANTE PRIVACIDAD:
- no guardar IP en texto plano
- guardar solo hash (sha256) con salt/pepper en servidor
- borrar huellas automaticamente (TTL / limpieza programada)


===============================================================================
4) ANONIMATO REAL (SIN PERFILES)
===============================================================================
No mostrar:
- userId
- nombre real
- avatar
- perfil clickable

Mostrar opcional:
- alias neutro por secreto/comentario: "Anon1234"

Regla:
- alias no debe permitir correlacion facil entre secretos distintos
- ideal: alias efimero por contenido (no global permanente)


===============================================================================
5) RANKING Y DESTACADOS (ENGANCHE VIRAL)
===============================================================================
Ranking base recomendado:
1. score = upVotesCount - downVotesCount
2. hotScore = score * pesoVotos + commentsCount * pesoComentarios - antiguedadHoras * decay
3. controversyScore = min(upVotesCount, downVotesCount) y totalVotes alto

Vistas destacadas:
- Top del dia:
  * ventana ultimas 24h
  * ordenar por hotScore desc

- Mas comentados:
  * ordenar por stats.commentsCount desc

- Mas votados:
  * ordenar por (upVotesCount + downVotesCount) desc

- Polemicos:
  * controversyScore desc
  * filtro minimo totalVotes >= N

Recomendacion tecnica:
- precalcular rankings en Cloud Functions (cron cada 5 min)
- guardar en _config/secret_rankings para lecturas rapidas


===============================================================================
6) MODERACION (CRITICA)
===============================================================================
Minimo obligatorio:
- boton "Reportar" en secretos y comentarios
- ocultado automatico por umbral de reportes
- panel admin para revisar y restaurar/bloquear

Guardrails sugeridos:
- bloqueo de doxxing basico (telefonos, mails, direcciones completas)
- bloqueo de insultos extremos / amenazas
- bloqueo de nombres propios en categoria "denuncia_light" (regla conservadora)

Politica sugerida:
- denuncias graves -> derivar a canal formal, no exponer datos personales
- logs de moderacion con trazabilidad (quien oculto, cuando, motivo)


===============================================================================
7) ANTI-SPAM Y ABUSO (MVP + HARDENING)
===============================================================================
Reglas iniciales:
- 1 secreto cada 30 minutos por fingerprintHash
- limite diario sugerido: 5 secretos / 24h por fingerprintHash
- 1 voto por secreto por fingerprintHash
- 1 reporte por secreto por fingerprintHash
- cooldown comentarios: 1 cada 20 segundos por fingerprintHash

Tecnico recomendado:
- crear secretos/votos/reportes via Callable Functions (no directo cliente)
- usar App Check + validaciones server-side
- rechazar textos duplicados exactos en ventana corta
- limitar links externos (o bloquearlos en MVP)


===============================================================================
8) REGLAS Y SEGURIDAD FIRESTORE (ADAPTACION)
===============================================================================
Ajustes necesarios en firestore.rules:
- habilitar lectura publica de module == "secrets" con moderation.status == "active"
- escritura de secretos/votos/reportes:
  * recomendado SOLO via Functions (cliente no escribe directo)
- comentarios anonimos:
  * permitir via Functions o reglas dedicadas con anti-abuso estricto

Nota:
- hoy tus reglas exigen auth para crear content/comments/likes.
- para "Secretos" sin login, conviene pasar por callable (backend decide).
