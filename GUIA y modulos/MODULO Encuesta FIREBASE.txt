Fecha: 2026-04-08
Proyecto: cdeluar-ddefc

RESUMEN
Se implemento el modulo de encuestas en tiempo real con:
- Vista publica en Inicio (encuesta destacada en tab Todos).
- Vista completa en tab Encuestas.
- Gestion staff (admin/colaborador) para crear, editar, finalizar, eliminar y activar/desactivar modulo.
- Voto seguro via Cloud Function callable (sin escritura directa de votos).
- Auto-completado de encuestas vencidas por scheduler.

El objetivo principal fue mantener UX rapida sin recargas globales y minimizar lecturas/escrituras innecesarias.


OBJETIVOS CUMPLIDOS
1) Visibilidad publica controlada
- En tab Todos se muestra solo 1 encuesta activa (destacada).
- Si hay varias activas, existe boton "Siguiente encuesta".
- En tab Encuestas se ve listado (activas + completadas).
- Si no hay encuestas activas, la tab "Encuestas" se oculta automaticamente.

2) Gestion staff completa
- Acceso por menu de usuario a "Gestion Encuestas" (solo staff).
- Crear/editar/eliminar/finalizar encuesta.
- Duracion por presets: 10 min, 30 min (default), 2 horas, 1 dia.
- Modulo encuestas habilitable/deshabilitable desde _config/modules.

3) Voto seguro y antifraude
- Voto solo autenticado.
- 1 voto por usuario por encuesta.
- Escritura de votos solo por callable submitSurveyVote.
- Validaciones de estado, expiracion, opciones validas y limite de seleccion.

4) Rendimiento
- Listener liviano para encuesta destacada.
- Sin recargar Home completo al votar (optimistic UI + reconciliacion snapshot).
- Opciones largas colapsadas (muestra max 3 + "Ver mas opciones").
- Scheduler backend para completar expiradas por lotes.


ARQUITECTURA IMPLEMENTADA
A) Configuracion de modulos
- Documento: _config/modules
- Campo canonico: surveys.enabled (boolean)
- Lectura en frontend por moduleStore listener.

B) Encuestas
- Coleccion: surveys/{surveyId}
- Campos principales:
  - question: string
  - description: string
  - status: active | inactive | completed
  - durationMinutes: 10 | 30 | 120 | 1440
  - expiresAt: timestamp
  - isMultipleChoice: boolean
  - maxVotesPerUser: number
  - totalVotes: number
  - options[]:
    - id: string (estable)
    - text: string
    - voteCount: number
    - active: boolean
  - createdBy, updatedBy, createdAt, updatedAt

C) Votos
- Coleccion: survey_votes/{surveyId}_{userId}
- Escritura: solo via Cloud Function callable.
- Lectura: owner y staff.

D) Funciones backend
- submitSurveyVote (callable, us-central1):
  - Valida auth, estado, expiracion, opciones y duplicado.
  - Usa transaction para escribir voto y sumar conteos atomicos.
- completeExpiredSurveys (scheduled cada 1 minuto):
  - Busca surveys activas vencidas (status=active, expiresAt<=now).
  - Actualiza status=completed por lotes.

PROBLEMAS REALES QUE HUBO Y SOLUCIONES APLICADAS
INCIDENTE 1 - Missing or insufficient permissions siendo admin
Sintoma:
- Usuario admin no podia ver/editar paneles.
Causa:
- Desalineacion entre chequeos UI, claims y rol.
Solucion:
- Se agrego helper unico de roles.
- Se reforzaron rules con fallback por super admin email/uid/claims.
- Se verifico usuario en Firebase Auth + users/{uid}.rol.

INCIDENTE 2 - Voto con error CORS/internal
Sintoma:
- Preflight CORS fallaba al votar.
Causa:
- submitSurveyVote no estaba desplegada en Functions.
Solucion:
- Deploy puntual de functions:submitSurveyVote.
- Verificacion posterior con functions:list (trigger callable activo).

INCIDENTE 3 - Encuestas activas no aparecian en feed principal
Sintoma:
- Encuesta activa existia pero no se veia en Todos.
Causa:
- Render solo en tab Encuestas.
Solucion:
- Se agrego bloque destacado en tab Todos con listener dedicado.

INCIDENTE 4 - En multiple opcion solo guardaba una respuesta
Sintoma:
- Usuario seleccionaba varias pero quedaba 1.
Causa:
- Configuracion posible con isMultipleChoice=true y maxVotesPerUser=1.
Solucion:
- Frontend y backend forzan minimo 2 cuando isMultipleChoice=true.

INCIDENTE 5 - Tab Encuestas molestaba sin contenido activo
Sintoma:
- Tab visible aunque no habia encuestas activas.
Solucion:
- Tab se oculta automaticamente si no hay activas.

CHECKLIST RAPIDO DE DIAGNOSTICO
A) "No aparece encuesta en Todos"
1) _config/modules.surveys.enabled == true
2) Existe al menos 1 survey con status=active y no vencida
3) Frontend actualizado (cache limpia)

B) "No deja votar"
1) Usuario autenticado
2) submitSurveyVote desplegada (functions:list)
3) Encuesta activa y no vencida
4) Opciones seleccionadas validas

C) "Error permisos staff"
1) users/{uid}.rol correcto
2) custom claims admin/superAdmin si aplica
3) relogin para refrescar token
4) firestore.rules desplegadas

D) "Scheduler no completa vencidas"
1) completeExpiredSurveys desplegada
2) Indice surveys(status+expiresAt) desplegado
3) expiresAt presente y valido en docs
